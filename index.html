Database Management System (DBMS)

1. Data

Data refers to raw facts or unprocessed information. It can be in the form of text, numbers, symbols, images, or sounds. On its own, data has no meaning, but once it is processed, it becomes useful information.

2. Data Types

Data types define the kind of value a variable can hold. Common data types in databases are:

  • Integer: Whole numbers
  • Float: Decimal numbers
  • Char: Single character
  • Varchar: Variable-length text
  • Date: Stores calendar date

3. Database

A database is an organized collection of data that allows easy access, management, and updating. It is used to store large amounts of information in a structured format.

4. Types of Databases

Databases are organized collections of data, and there are different types based on their structure and how they manage data. The main types of databases include:

1. Relational Database

  • Structure: Uses tables (also called relations) consisting of rows and columns.
  • How it works: Each table represents an entity, and relationships between tables are defined using keys.
  • Example: MySQL, Oracle, Microsoft SQL Server.
  • Use case: Suitable for applications needing structured data with complex queries like banking systems, online stores.

2. Hierarchical Database

  • Structure: Data is organized in a tree-like structure with a single root and multiple levels of parent-child relationships.
  • How it works: Each parent record can have multiple child records but each child has only one parent.
  • Example: IBM's IMS.
  • Use case: Useful for applications like file systems or organizational charts.

3. Network Database

  • Structure: Uses a graph structure where each record can have multiple parent and child records (many-to-many relationships).
  • How it works: More flexible than hierarchical; allows complex relationships.
  • Example: Integrated Data Store (IDS).
  • Use case: Suitable for telecommunications and complex data models.

4. Object-Oriented Database

  • Structure: Stores data as objects, similar to object-oriented programming concepts.
  • How it works: Supports classes, inheritance, and encapsulation allowing data and behavior to be stored together.
  • Example: db4o, ObjectDB.
  • Use case: Useful in applications that require complex data types like CAD, multimedia, or scientific data.

5. NoSQL Database (Non-Relational)

  • Structure: Does not rely on tables. Can be document-based, key-value pairs, wide-column stores, or graph databases.
  • How it works: Designed for scalability and flexible schema.
  • Example: MongoDB (document), Redis (key-value), Cassandra (wide-column).
  • Use case: Big data, real-time web applications, social networks.

5. Software & Domain-Specific Software

Software is a set of programs used to operate computers and execute specific tasks. Domain-specific software is designed for a particular field or industry.

  • Accounting System: Manages financial data and transactions.
  • Attendance System: Tracks presence and absence of individuals.

6. Database Management System (DBMS)

A Database Management System (DBMS) is a software application that enables users to define, create, maintain, and control access to databases. It acts as an intermediary between users and the database, ensuring efficient and secure data management.

Key features of DBMS:

  • Data Organization: Organizes data in structured formats for easy access and manipulation.
  • Data Integrity: Maintains accuracy and consistency of data through rules and constraints.
  • Data Security: Protects data using authentication, access controls, and permissions.
  • Data Sharing: Allows multiple users to access and work on data concurrently without conflict.
  • Backup and Recovery: Automatically backs up data and restores it in case of failure.
  • Query Processing: Supports powerful query languages like SQL for data retrieval and modification.
  • Transaction Management: Ensures that all database operations are completed fully or not at all, preserving data consistency.
  • Types of DBMS: Includes relational, hierarchical, network, object-oriented, and NoSQL systems.

Popular DBMS software includes MySQL, Oracle Database, Microsoft SQL Server, PostgreSQL, and MongoDB.

7. Database Tools

Tools help manage and interact with databases. Examples include:

  • phpMyAdmin
  • Oracle SQL Developer
  • HeidiSQL

8. Server

A server is a system that provides resources and services to other computers (clients) over a network. Servers come in various types depending on their role:

  • Database Server: Hosts databases and processes data requests (e.g., MySQL Server, Oracle Database Server).
  • Web Server: Serves web pages to clients (e.g., Apache HTTP Server, Nginx).
  • File Server: Stores and manages files shared over a network (e.g., Windows File Server, Samba).
  • Mail Server: Manages sending and receiving emails (e.g., Microsoft Exchange Server, Postfix).
  • Application Server: Runs specific applications for clients (e.g., IBM WebSphere, JBoss).

9. SQL Commands

SQL (Structured Query Language) is used to communicate with databases. It includes different types of commands:

Data Definition Language (DDL)

Commands that define or modify the structure of database objects.

  • CREATE - Creates a new table or database.
    Example:
    CREATE TABLE Students (StudentID INT PRIMARY KEY, Name VARCHAR(50), Age INT);
  • ALTER - Modifies an existing table.
    Example:
    ALTER TABLE Students ADD COLUMN Email VARCHAR(100);
  • DROP - Deletes a table or database.
    Example:
    DROP TABLE Students;

Data Manipulation Language (DML)

Commands used to manipulate data within tables.

  • INSERT - Adds new records.
    Example:
    INSERT INTO Students (StudentID, Name, Age) VALUES (1, 'Areej', 18);
  • UPDATE - Changes existing records.
    Example:
    UPDATE Students SET Age = 19 WHERE StudentID = 1;
  • DELETE - Removes records.
    Example:
    DELETE FROM Students WHERE StudentID = 1;

Data Query Language (DQL)

Commands used to query or retrieve data.

  • SELECT - Fetches data from tables.
    Example:
    SELECT Name, Age FROM Students WHERE Age > 17;

Data Control Language (DCL)

Commands that control access to the database.

  • GRANT - Gives privileges to users.
    Example:
    GRANT SELECT ON Students TO 'user1';
  • REVOKE - Removes privileges.
    Example:
    REVOKE SELECT ON Students FROM 'user1';

10. CRUD Operations

CRUD stands for Create, Read, Update, and Delete, which are the four basic operations used to manage data in a database. These operations allow users to add new information (Create), retrieve or view existing information (Read), modify or change existing data (Update), and remove unwanted or obsolete data (Delete). Together, CRUD operations form the foundation of working with databases, enabling users to interact with and maintain data effectively. For example, when you register for an account on a website, the system creates a new record in the database. When you log in and view your profile, the system reads your data. If you update your contact information, the system performs an update operation, and if you decide to delete your account, the system deletes the record. These operations are typically implemented using SQL commands such as INSERT (Create), SELECT (Read), UPDATE (Update), and DELETE (Delete).

CRUD operations include:

  • Create: Adding new data or records to the database.
  • Read: Retrieving or viewing existing data.
  • Update: Modifying or changing existing data.
  • Delete: Removing data that is no longer needed.

11. Keys in Database

Keys are important attributes in a database that help uniquely identify records and establish relationships between tables. The primary key ensures each record is unique, while foreign keys link tables together. Candidate keys are possible choices for primary keys, and composite keys combine multiple fields to create a unique identifier.

Types of keys:

  • Primary Key: Uniquely identifies each record in a table.
  • Foreign Key: Links a field in one table to the primary key in another table.
  • Candidate Key: Any field or combination of fields that could serve as a primary key.
  • Composite Key: A key made up of two or more columns to uniquely identify a record.

12. Constraints on Database

Constraints are rules applied to database columns to ensure the validity, accuracy, and integrity of the data stored. They prevent incorrect data entry and maintain relationships between tables.

Common constraints:

  • NOT NULL: Ensures a column cannot have empty (null) values.
  • UNIQUE: Ensures all values in a column are different.
  • PRIMARY KEY: Combination of NOT NULL and UNIQUE, uniquely identifies each record.
  • FOREIGN KEY: Ensures the value in a column matches a value in another tableâ€™s primary key, maintaining referential integrity.
  • CHECK: Ensures that values meet a specific condition or range.

13. Anomalies in Database

Anomalies occur due to poor database design and can cause inconsistent or incorrect data during data operations such as insertions, deletions, or updates.

Types of anomalies:

  • Insertion Anomaly: Difficulty inserting data because some data is missing or dependent data does not exist.
  • Deletion Anomaly: Loss of important data unintentionally when other data is deleted.
  • Update Anomaly: Data inconsistencies that happen when redundant data is updated partially or incorrectly.

14. Normalization in Database

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller related tables and defining relationships between them.

Common normal forms:

  • 1NF (First Normal Form): Remove repeating groups; each column should hold atomic values.
  • 2NF (Second Normal Form): Remove partial dependency; every non-key attribute must depend on the whole primary key.
  • 3NF (Third Normal Form): Remove transitive dependency; non-key attributes should depend only on the primary key.

15. Database Security

Database security involves protecting the database from unauthorized access, misuse, or threats to ensure data confidentiality, integrity, and availability. It includes various measures to safeguard the data against accidental or malicious damage.

Key aspects of database security:

  • Access Control: Restricts who can view or manipulate data through user roles and permissions.
  • Authentication: Verifies the identity of users before granting access.
  • Encryption: Encodes data so unauthorized users cannot read it even if accessed.
  • Backup and Recovery: Maintains copies of data to restore in case of data loss or corruption.
  • Audit Trails: Logs activities in the database to monitor and detect suspicious behavior.
  • Physical Security: Protects hardware and physical storage from damage or theft.
